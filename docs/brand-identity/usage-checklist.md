# Brand Usage Checklist

Use this before publishing decks, one-pagers, landing pages, and pilot docs.

## Visual checks

- Primary logo selected for background (dark/light variant used correctly)
- Clear space preserved around logo
- Colors limited to approved palette
- Typography matches package hierarchy
- Visuals feel field-practical, not speculative/futuristic

## Message checks

- Claims are evidence-based and realistic
- Language is direct and operational
- Audience value is explicit (owner, manager, crew)
- “No manual entry” and “verified truth” are framed accurately

## Content checks

- Pilot state is labeled honestly when appropriate
- Hardware references match technical docs
- Pricing/context avoids over-promising
- Terminology is consistent: badge, equipment tag, cloud event, dashboard
